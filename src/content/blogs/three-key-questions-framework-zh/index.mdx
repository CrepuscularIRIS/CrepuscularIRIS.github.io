以会议速度阅读 ML 论文会产生一种特定的认知债务：你积累的声称改进超过了你能评估它们的速度。一个新的残差变体。一个注意力机制的修改。一个路由机制。每一个都附带着一张表格，显示它比四个标准基准高出 1-2%。

大多数这些声称无法干净地复现。改进在论文的实验设置中是真实的——但这个设置有着隐藏的自由度，解释了那个差距。架构的贡献小于作者们所相信的，而且贡献来自于他们所描述的机制之外的地方。

三个问题被证明在区分信号与噪声时始终有用。它们适用于任何架构提案，但当声称是对现有架构的修改（残差连接、注意力机制、归一化策略）而非全新的范式时，效果最为锋利。

## 问题一：这是改变权重分配还是改变被建模的对象？

第一个轴将**连接权重修改**与**建模对象变更**区分开来。这个区别很重要，因为它们有着根本不同的失败模式和不同的上限。

**连接权重修改**改变了信息在现有计算图中被聚合、路由或组合的方式。输入和输出保持不变；变换改变了。例如：

- 注意力残差（固定权重 → 对层历史的 Softmax 加权聚合）
- 专家混合路由（密集 FFN → 稀疏专家选择）
- 可学习层缩放（固定单位残差权重 → 每层增益参数）

**建模对象变更**改变了网络被要求表示的内容。计算结构可能看起来相似，但模型现在捕捉的是根本不同的分布假设。例如：

- 变分自编码器（直接密度估计 → 带显式先验的隐变量建模）
- 神经 ODE（离散层堆叠 → 深度空间中的连续动力学）
- 扩散模型（直接生成 → 分数函数估计和反向 SDE）

实用测试：如果基线模型可以在测试时用正确的学习权重来近似提议模型的行为——那你看到的就是连接权重修改。如果基线模型*原则上无法*表示提议所表示的内容，那你看到的就是建模对象变更。

```python
# 连接权重修改：基线可以在初始化时表达这个
# 注意力残差使用 Softmax 权重 α_i 对层历史 h_i
# 在初始化时，α_i ≈ 均匀 → 等同于标准残差 / L
# 在收敛时，α_i = f(content) → 超出基线表达力，但
#     过渡是平滑的，基线"差不多做到了"

h_attn_residual = sum(alpha_i * h_i for i in range(current_layer))

# 建模对象变更：基线无法表达这个分布
# VAE 建模 p(x) = ∫ p(x|z)p(z)dz — 需要显式隐结构
# 标准自编码器建模 p(x|enc(x)) — 无法近似积分
# 这些不是通过权重变化关联的；它们建模的是不同的事物

z_mean, z_logvar = encoder(x)
z = reparameterize(z_mean, z_logvar)  # 需要随机计算图
x_recon = decoder(z)
kl_loss = -0.5 * sum(1 + z_logvar - z_mean**2 - exp(z_logvar))
```

**为什么这在评估中很重要**：正确评估连接权重修改的方法是询问学到的权重是否真的与初始化不同——以及差值是否为正。如果你训练注意力残差并发现学到的 α_i 大致是均匀的，那么模型没有在使用提议的机制，即使基准有改善。（参数数量增加带来的基准改善是这类修改最常见的混淆因素。）

建模对象变更需要不同的评估方式：你需要证明新的建模假设*对这个领域是正确的*，而不仅仅是模型达到了更好的指标。VAE 的价值来自于学习一个有意义的隐空间，而不仅仅是重建准确率。

**怀疑论者从问题一的结论**：带有基准改善的连接权重修改是一个弱声称，除非论文表明 (a) 学到的权重实际上不同于均匀初始化，(b) 改善不能用额外的参数来解释。

## 问题二：这是块级还是承诺了连续性？

第二个问题问的是提议机制在**离散化谱**上的位置：从完全块局部的（在固定计算组内操作，块之外没有结构）到完全连续的（在没有离散边界的平滑流形上操作）。

这很重要，因为大多数"新颖"架构在这个谱上是隐含的妥协——而论文很少告诉你它们落在那个位置时放弃了什么。

**块局部机制**在固定大小的层块、token 块或时间步块内操作。"块"是一个设计选择：4 层、8 个 token、512ms 的音频。在块内，机制可以任意复杂。跨块交互通过固定残差或拼接完成。

例如：Transformer 块（在上下文窗口内的注意力，基础 GPT 中没有跨窗口交互）、块注意力残差（在每个 B 层块*内*进行注意力加权聚合，跨块用固定残差）、滑动窗口注意力。

**连续机制**承诺在相关维度上的平滑变化。神经 ODE 将深度参数化为连续变量；记忆增强架构将时间视为连续信号；CDMN 风格的提案将层深度视为学习流形中的坐标。

```python
# 块级：交互被 block_size 限制
class BlockAttnRes(nn.Module):
    def __init__(self, block_size: int = 4):
        self.block_size = block_size  # 硬边界
    
    def forward(self, h_current: torch.Tensor, 
                history: list[torch.Tensor],
                layer_idx: int) -> torch.Tensor:
        # 只能在当前块内注意力 — 硬约束
        block_start = (layer_idx // self.block_size) * self.block_size
        block_history = history[block_start:layer_idx]  # 最多 O(B) 项
        
        if not block_history:
            return h_current
        
        # 块内注意力是任意的；边界是固定的
        return self._attend(h_current, torch.stack(block_history, dim=0))
