## 核心框架

本文系统梳理深度学习架构演进的**三条核心路线**，分别对应三种不同的"建模哲学"：

| 路线 | 核心思想 | 代表工作 | 关键问题 |
|------|----------|----------|----------|
| **路线 A** | 挑战 Residual | Kimi Attention Residuals | 连接权重 vs 建模对象？ |
| **路线 B** | Orchestration / Query-based | BLIP-2, Perceiver IO | Block 级妥协 vs 连续化？ |
| **路线 C** | 连续潜变量 | VAE, β-VAE | 架构灵感 vs 训练工程？ |

---

## 路线 A：直接挑战 Residual

### 核心洞察

**经典 Residual** 的本质是"训练可优化性的工程支架"：
```
h_{l+1} = h_l + f(h_l)  # 固定累加，权重隐式为 1
```

**Attention Residuals** 将其改造为"检索式编排"：
```
h_{l+1} = Σ α_i · h_i  # 学习权重，动态选择
```

### 关键区分

| 维度 | 经典 Residual | Attention Residuals |
|------|---------------|---------------------|
| **权重性质** | 固定 (implicit 1) | 学习 (softmax 归一化) |
| **连接方式** | 仅前一层 | 任意前层 |
| **优化目标** | 梯度流畅 | 动态信息路由 |
| **哲学定位** | 工程妥协 | 建模增强 |

### 材料定位

**Kimi Attention Residuals** (arXiv)
- 核心贡献：用 depth-wise attention 替换固定累加
- 解决痛点：PreNorm 中 hidden state 膨胀、单层贡献稀释
- 技术细节：待阅读原始论文

**关键问题**：
- 改的是"连接权重"还是"建模对象"？→ **权重形式，但影响建模**
- Block 级还是连续化？→ **Block 级妥协**
- 架构灵感还是产品披露？→ **架构灵感**

---

## 路线 B：Orchestration / Query-based Selection

### 核心洞察

不是"把信息全拼上去"，而是"用 query 去筛选/编排"。

### 代表工作

**1. BLIP-2 / Q-Former**
- 机制：轻量级查询 Transformer，从图像提取固定数量特征
- 哲学：信息筛选优于信息堆砌
- 关键：Query 机制实现选择性注意力

**2. Perceiver IO**
- 机制：先投射到 latent，再用 query 读出
- 哲学："memory coordinate + selective retrieval"
- 关键：与"深度即记忆地址"思想呼应

### 与 CDMN 的关联

| 概念 | Perceiver IO | CDMN (我的框架) |
|------|--------------|-----------------|
| 记忆表示 | Latent array | Depth landmarks |
| 查询机制 | Cross-attention | RoPE phase retrieval |
| 选择性 | Top-k latent slots | Top-k depth positions |
| 连续化 | Discrete latent | Continuous depth flow |

**关键问题**：
- 改的是"连接权重"还是"建模对象"？→ **建模对象（从拼接改为查询）**
- Block 级还是连续化？→ **更接近连续化**
- 架构灵感还是产品披露？→ **架构灵感**

## 路线 C：连续潜变量妥协

### 核心洞察

VAE 不是在解决 residual/attention 问题，而是在处理另一层面的"建模哲学"：

> **如何用可训练的妥协，把连续潜变量结构塞回离散优化流程？**

### 代表工作

**1. VAE (Auto-Encoding Variational Bayes)**
- 原始论文：Kingma & Welling, 2013
- 核心： encoder-decoder + KL 正则的连续潜空间
- 妥协：重参数化技巧、KL 散度平衡

**2. β-VAE**
- 改进：引入 β 参数控制解耦程度
- 洞察：表示学习需要在重构质量和解耦性之间权衡

### 与架构演进的关联

| 路线 | 离散-连续光谱 | 核心妥协 |
|------|---------------|----------|
| Residual | 离散 | 梯度流畅性 |
| Attention Residuals | 离散 | 动态权重选择 |
| Orchestration | 半离散 | Query-based 离散化 |
| VAE | 连续 (编码为离散) | 训练稳定性 |
| **CDMN (目标)** | **连续** | **相位编码离散化** |

**关键问题**：
- 改的是"连接权重"还是"建模对象"？→ **建模对象（潜变量结构）**
- Block 级还是连续化？→ **连续化建模，离散化训练**
- 架构灵感还是产品披露？→ **架构灵感（训练哲学）**

## 统一视角：三个关键问题

筛选任何新材料时，问自己这三个问题：

### 1. 改"连接权重"还是改"建模对象"？

| 类型 | 例子 | 影响深度 |
|------|------|----------|
| 连接权重 | Attention Residuals | 浅层优化 |
| 建模对象 | Orchestration, VAE | 深层范式 |

### 2. Block 级妥协还是连续化？

| 类型 | 例子 | 理想程度 |
|------|------|----------|
| Block 级 | Attention Residuals, MoE | 工程可行 |
| 半连续 | Perceiver IO, Q-Former | 探索中 |
| 连续 | Neural ODE (理想) | 训练昂贵 |
| **目标** | **CDMN** | **待验证** |

### 3. 架构灵感还是产品/安全披露？

| 类型 | 信息密度 | 研究价值 |
|------|----------|----------|
| 架构灵感 | arXiv 论文, 开源代码 | 高 |
| 产品披露 | 发布页, benchmark 数字 | 中 |
| 安全披露 | System Card, 政策文档 | 低（对你当前研究） |
