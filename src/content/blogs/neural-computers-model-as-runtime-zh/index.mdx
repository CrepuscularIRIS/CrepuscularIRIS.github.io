每台计算机——从大型机到 GPU——都遵循冯·诺依曼架构：存储、处理和 I/O 被分离到通过总线连接的独立子系统中。程序驻留在内存里，处理器获取指令、执行指令、写回结果。数据沿定义好的通道在这些组件间流动。

然而，神经网络虽然在冯·诺依曼硬件上运行，但其内部运作方式截然不同。Transformer 的前向传播不从存储的程序中获取指令，而是在一次可微分扫描中，将学到的变换作用于输入数据。"程序"是权重，"内存"是激活值，"I/O"是分词器和输出头。这些并非独立的子系统——它们被统一到一个单一的computational graph中。

Meta AI 和 KAUST 提出的神经计算机（NC）提案将这一观察推向逻辑极致：假如我们设计的架构让模型不只是"类似于"运行时——而是*就是*运行时呢？计算、内存和 I/O 被明确内化到学习到的参数中，"模型"与"运行模型的系统"之间的传统区分就此消解？

这不是比喻，而是有具体机制支撑的明确架构提案。

## 外部状态的问题

当前大语言模型与世界的交互界面很窄：文本进、文本出，辅以触及模型边界之外的可选工具调用。当模型需要在多轮对话中记住信息时，它依赖外部内存——追加到上下文窗口的对话历史、从向量数据库中检索的 RAG，或者由推理服务器维护的显式 KV 缓存。

这种架构有三个结构性弱点：

**序列化开销。** 每次模型与外部状态交互，都需要将内部表示转换为文本（或 token），穿过接口边界，再转换回来。模型对"用户认证状态"的内部表示是丰富的高维激活模式；外部表示是一个 JSON blob。每一次边界穿越都丢失信息。

**一致性脆弱。** 外部内存系统不与模型的已学习表示共享空间。向量数据库存储的是 embedding，但这些 embedding 独立于模型当前推理状态计算得出。当模型检索"相关"文档时，相关性由 embedding 空间的余弦相似度决定——而不是由模型内部评估该信息对当前计算的实际帮助程度。检索步骤与推理步骤从根本上脱节。

**扩展不连续。** 向外部内存系统增加更多上下文，在检索时需要线性（或更差）的计算，但每个检索项的边际效用迅速递减。模型利用检索信息的能力取决于它在上下文窗口中的位置、相关信息密度相对于噪声的比值，以及模型的注意力容量。这些由架构决定，不由你配置的内存量决定。

神经计算机通过将内存变成学习到的内部组件来解决这些问题——它是模型参数空间的一部分，而非外部系统。

## 架构：三个内部子系统

NC 架构用三个学习到的子系统取代了传统的模型/基础设施切分：

### 计算核心

计算核心是一个改进的 transformer 堆栈，其中每层既能执行标准的 token 级变换，也能执行元计算——对模型自身内部状态的操作。标准层处理输入序列。元计算层检查并修改前几层的激活模式，有效实现学习到的控制流。

```python
@dataclass(frozen=True)
class NCLayerConfig:
    """Configuration for a Neural Computer layer."""
    hidden_dim: int
    num_heads: int
    layer_type: str  # "standard" | "meta" | "memory_read" | "memory_write"
    memory_slots: int = 0

class MetaComputationLayer(nn.Module):
    """Layer that operates on the model's own activation state."""

    def __init__(self, cfg: NCLayerConfig):
        super().__init__()
        self.state_inspector = nn.MultiheadAttention(
            embed_dim=cfg.hidden_dim,
            num_heads=cfg.num_heads,
        )
        self.state_modifier = nn.Sequential(
            nn.Linear(cfg.hidden_dim, cfg.hidden_dim * 4),
            nn.GELU(),
            nn.Linear(cfg.hidden_dim * 4, cfg.hidden_dim),
        )
        self.gate = nn.Linear(cfg.hidden_dim, cfg.hidden_dim)

    def forward(
        self,
        x: torch.Tensor,
        layer_states: list[torch.Tensor],
    ) -> torch.Tensor:
        # Attend over previous layer activations
        # Each state: (batch, seq_len, hidden_dim)
        # Concatenate along seq_len to form a flat key/value sequence
        stacked = torch.cat(layer_states, dim=1)  # (batch, num_layers*seq, dim)
        inspected, _ = self.state_inspector(
            query=x, key=stacked, value=stacked,
        )
        # Compute modification signal
        modification = self.state_modifier(inspected)
        # Gated update — model learns when to apply meta-computation
        gate_signal = torch.sigmoid(self.gate(x))
        return x + gate_signal * modification
```

门控机制至关重要。没有它，元计算层会在每一次前向传播中修改激活，导致不稳定。学习到的门意味着模型可以在标准处理足够时选择跳过元计算——有效地实现了条件计算，而无需显式分支。

### 持久内存

内存子系统使用学习到的内存槽——跨前向传播持久存在的固定大小参数张量，通过门控写入更新。与 KV 缓存（存储投影后的 key-value 表示，随序列长度线性增长）不同，持久内存将*压缩的、学习到的表示*存储在固定容量的缓冲区中。

读取机制使用可微分注意力；写入机制使用截断反向传播（分离梯度）来保证训练稳定性——权衡原因见下文：

```python
class PersistentMemory(nn.Module):
    """Fixed-capacity learned memory that persists across forward passes."""

    def __init__(self, num_slots: int, slot_dim: int):
        super().__init__()
        self.slots = nn.Parameter(torch.randn(num_slots, slot_dim) * 0.02)
        self.write_head = nn.MultiheadAttention(slot_dim, num_heads=8)
        self.read_head = nn.MultiheadAttention(slot_dim, num_heads=8)
        self.write_gate = nn.Linear(slot_dim, 1)

    def read(self, query: torch.Tensor) -> torch.Tensor:
        """Content-addressable read from memory slots."""
        result, attention_weights = self.read_head(
            query=query,
            key=self.slots.unsqueeze(0).expand(query.size(0), -1, -1),
            value=self.slots.unsqueeze(0).expand(query.size(0), -1, -1),
        )
        return result

    def write(self, content: torch.Tensor) -> None:
        """Detached write — updates slots via attention with truncated BPTT."""
        with torch.no_grad():
            gate = torch.sigmoid(self.write_gate(content)).mean()
            if gate > 0.5:  # Only write when gate activates
                update, _ = self.write_head(
                    query=self.slots.unsqueeze(0),
                    key=content,
                    value=content,
                )
                self.slots.data = (
                    0.9 * self.slots.data + 0.1 * update.squeeze(0)
                )
```

这是现代可微分版的神经图灵机内存——但在规模上有两个重要差异。首先，写入门防止灾难性覆写。其次，指数移动平均更新意味着旧记忆优雅衰减，而非被突然替换。

### I/O 接口

I/O 子系统处理与外部世界的交互，但不再将 I/O 视为边界穿越（序列化 → 传输 → 反序列化），而是将外部信号视为学习系统的连续输入。

工具调用、传感器读数和用户输入通过学习到的编码器投影到模型的内部表示空间。模型收到的不是"网络搜索结果文本"——而是与它的内部激活共享同一表示空间的搜索结果连续 embedding。

这消除了有损的文本序列化步骤。模型可以用与内部推理相同的机制来关注工具结果，因为表示处于同一空间中。

## 为什么这个架构重要

NC 提案不只是学术好奇。它解决了当前 AI 系统的三个现实问题：

**整合税。** 每次向 AI 系统添加工具、内存系统或外部组件，你都在缴税：需要在模型的内部表示与外部系统的数据格式之间转换的overhead。NC 将这些组件内化，完全消除了整合税。模型的"工具"是学习到的子系统，而非外部服务。

**可组合性问题。** 当前agent架构通过链式组合——一个模型/工具的输出作为下一个的输入。链中每个环节都引入延迟和信息损失。NC 通过集成组合——不同能力是同一前向传播中的不同层，共享激活值和梯度。

**训练-推理差距。** 标准模型在静态数据集上训练，却在带有工具和内存的动态环境中部署。模型需要在推理时通过上下文学习或 RLHF 弄清如何使用工具，因为工具不是训练架构的一部分。NC 在训练时激活其内存和 I/O 子系统，因此工具使用从一开始就是学到的能力。

## 难题：训练稳定性

让这一切运转需要解决当前 transformer 训练不面对的训练稳定性问题。当内存槽在前向传播中更新时，反向传播的梯度计算必须考虑每个时间步的内存状态。这与早期循环神经网络面临的挑战相同——但在 transformer 规模上。

提议的解决方案使用分离的内存梯度：前向传播更新内存，但反向传播在每个时间步将内存状态视为固定。这是*有偏的*——它不计算关于内存动态的真正梯度——但它是稳定的。偏差可接受，因为内存写入门学会做出小的增量更新，所以真正梯度与分离梯度保持接近。

```python
class NCTrainingStep:
    """Simplified training step for a Neural Computer."""

    def __init__(self, model: NeuralComputer, optimizer: torch.optim.Optimizer):
        self.model = model
        self.optimizer = optimizer

    def step(self, batch: dict[str, torch.Tensor]) -> float:
        self.optimizer.zero_grad()

        # Forward pass updates memory in-place
        output = self.model(
            input_ids=batch["input_ids"],
            attention_mask=batch["attention_mask"],
        )

        loss = F.cross_entropy(
            output.logits.view(-1, output.logits.size(-1)),
            batch["labels"].view(-1),
        )

        # Backward pass uses detached memory states
        loss.backward()

        # Clip gradients — critical for stability with memory updates
        torch.nn.utils.clip_grad_norm_(self.model.parameters(), max_norm=1.0)

        self.optimizer.step()
        return loss.item()
```

梯度裁剪不是可选项。没有它，元计算层通过状态检查机制放大梯度，导致训练在几百步内发散。

## 这改变了什么（和没改变什么）

如果 NC 架构在规模上work，它改变了我们对模型与系统边界的思考方式。今天，部署 AI 系统意味着在模型*周围*构建基础设施——检索管道、工具服务器、内存存储、编排层。NC 会将这些基础设施的大部分内化到模型本身中。

没改变的是：学习系统的根本局限。内部化了自身内存的模型仍然无法存储它没有训练过或推理时没遇到过的信息。持久内存是有限的、有损的。I/O 子系统只能通过其学习到的编码器与外部世界接口——它不能自发发展出新的输入模态。

真正的问题不是这个架构能否取代今天的 transformer 加基础设施栈。当前系统的整合税和可组合性overhead是否大到值得端到端学习替代方案的训练复杂度？对于延迟和一致性比模块化更重要的应用——实时agent系统、具身机器人、持续监控——答案可能是肯定的。

## 实践启示

对于从业者，NC 架构暗示了一个值得在传统系统中采纳的设计原则：最小化边界穿越。每次你的 AI 系统将内部状态序列化为文本、通过 API 传递、另一端反序列化时，你都在支付整合税。通过共享 embedding 空间、统一的推理管道、或模型与基础设施之间更紧密的耦合来减少这些边界——这正是 NC 架构指向的方向。

NC 论文还澄清了 AI 系统"端到端学习"的含义。当前系统在模型边界内是端到端的（输入 token 到输出 token），但在系统边界上是模块化的（模型 + 检索 + 工具 + 内存）。真正的端到端学习将跨越整个系统——包括目前位于模型之外的基础设施组件。在生产规模上这是否可行仍是开放问题，但架构方向是清晰的。
