大多数深度学习从业者把架构当作黑箱来学习。ResNet 能跑通，就用 ResNet。Transformer 能跑通，就用 Transformer。这对工程来说够了，但对研究来说是错误的立场。理解一个架构*为什么*有效——以及它隐含假设了什么——是判断何时应该放弃它的必要条件。

以下三个维度构成了一套系统性的框架，用于从第一性原理评估任何架构提案。它们同样适用于评估现有架构（为什么 ResNet 在任务 X 上失败了？）和设计新架构（我的任务施加了什么约束？）。这个框架产生于对残差连接变体的分析，但维度本身是通用的。

## 维度一：交互顺序

**核心问题**：这个架构能够表示的最高阶交互是什么？

神经网络通过元素之间的交互来处理信息——token、通道、层、模态。交互的阶数决定了表达能力：一阶分别处理每个元素，二阶处理成对关系，高阶处理任意子集。

这很重要，因为任务本身具有内在的交互阶数。场景理解需要理解"桌布 + 盘子 + 餐具 + 食物"共同定义了一个用餐场景——这是一种高阶关系，无法在不损失信息的情况下分解为成对交互。序列建模需要"not ... but ..."是非相邻 token 之间的依赖——这种关系固定窗口架构可能无法捕捉。

常见架构的交互阶数：

| 架构 | 交互阶数 | 机制 |
|---|---|---|
| MLP | 一阶 | 每个 token 独立处理 |
| CNN | 二阶 | 固定卷积核内的局部成对关系 |
| 标准 ResNet | 二阶 | 仅相邻层之间的交互 |
| Transformer（自注意力） | 二阶 | 每个头的全对点积 |
| 超图神经网络 | 高阶 | 超边连接任意子集 |

实践含义：在阶数 $k$ 上运行的架构需要 $O(\text{depth})$ 来综合阶数 $> k$ 的信息。一个需要综合 4-way 交互的 transformer 必须链式叠加三个二阶注意力操作，这引入了梯度路径长度约束，使综合变得隐式而非直接。

```python
import torch
import torch.nn as nn
from typing import Optional

class SecondOrderAttention(nn.Module):
    """标准自注意力：全对二阶交互。"""
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # x: (B, T, D)
        # Attention(Q, K, V): token i 关注 token j — 成对关系
        # 一个 4-way 交互至少需要 3 层堆叠的注意力
        q, k, v = x, x, x
        scores = torch.bmm(q, k.transpose(1, 2)) / (x.shape[-1] ** 0.5)
        return torch.bmm(torch.softmax(scores, dim=-1), v)


class HigherOrderHyperedgeAttention(nn.Module):
    """
    基于超边的注意力：显式高阶交互。
    
    一个超边将 k 个 token 分组并产生一个输出，
    表示它们的联合交互——不分解为成对关系。
    这就是"场景"和"物体关系"的区别。
    """
    def __init__(self, d_model: int, n_hyperedges: int = 16, edge_size: int = 4):
        super().__init__()
        self.edge_size = edge_size
        # 可学习的超边成员权重
        self.edge_membership = nn.Parameter(torch.randn(n_hyperedges, d_model))
        # 对于每个超边，将成员 token 聚合为一个表示
        self.edge_proj = nn.Linear(d_model, d_model)
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        x: (B, T, D)
        对于每个超边 e，计算：
          membership_score[e, t] = token t 对 e 的亲和度的 softmax
          edge_repr[e] = 成员 token 表示的加权和
        然后每个 token 从它所属的所有超边聚合信息。
        """
        B, T, D = x.shape
        
        # (B, T, n_edges): 每个 token 属于每个超边的程度
        membership = torch.softmax(
            torch.einsum("btd,ed->bte", x, self.edge_membership) / (D ** 0.5),
            dim=1  # 在 token 上 softmax → 稀疏的边成员
        )
        
        # (B, n_edges, D): 边表示 = 成员 token 的加权和
        edge_repr = torch.einsum("bte,btd->bed", membership, x)
        edge_repr = self.edge_proj(edge_repr)  # 学习到的变换
        
        # (B, T, D): 每个 token 从所有它所属的边聚合
        out = torch.einsum("bte,bed->btd", membership, edge_repr)
        return out + x  # 残差连接
```

**诊断问题**：如果你的任务要求模型基于元素之间的 4-way 或 5-way 联合交互来做决策，你的架构是否能够在不产生不实际的深度的情况下表示这种关系？

## 维度二：均匀性假设

**核心问题**：这个架构在哪些维度上均匀对待所有元素，这种处理对你的任务是否正确？

均匀性是假设沿某个维度的所有元素都是等价的，应该接受同等的计算处理。这是一个强有力的归纳偏置，通常是错的。六个经常被错误假设均匀性的维度：

**深度均匀性**：标准残差连接假设每个层对最终表示的贡献相等：

$$h_{l+1} = h_l + f(h_l) \quad \Rightarrow \quad h_L = h_0 + \sum_{l=0}^{L-1} f(h_l)$$

每个层贡献的隐式权重是 1。在实践中，不同深度编码定性不同的信息（浅层是低级特征，深层是高级语义），有些层比其他层贡献大得多。这被层丢弃实验经验性地证明：大型模型在移除 30-40% 的层后通常仍然可用，如果所有层贡献相等则不可能如此。

**宽度均匀性**：标准卷积将所有通道同等对待——每个输出通道是所有输入通道的学习线性组合，计算优先级相同。但通道在训练过程中会特化；一些编码任务相关特征，一些变得近乎冗余。通道注意力（SE-Networks、ECA-Net）解决了这个问题，但底层架构从一开始就是均匀的。

**空间均匀性**：3×3 卷积无论图像内容如何都采样固定网格。高信息密度区域（物体边界、细粒度纹理）与背景接受相同数量的采样点。可变形卷积解决了这个问题；标准卷积没有。

**时间均匀性**：标准 transformer 注意力没有内置的近因偏差——没有内容相关的线索时，所有位置接受等效加权。架构中没有"近期上下文比远期上下文更相关"的先验；这必须完全从数据中学习。

```python
class NonUniformDepthAggregation(nn.Module):
    """
    可学习的深度聚合：每个层获得学习到的权重，
    而不是标准残差的固定权重 1。
    
    这是注意力残差（Attention Residuals，Moonshot AI，2026）的核心思想：
    不是计算 h_{L} = h_0 + sum(f(h_l))，而是计算
    h_{L} = sum(alpha_l * h_l)，其中 alpha_l 是学习到的或内容依赖的。
    """
    def __init__(self, n_layers: int, d_model: int):
        super().__init__()
        # 内容依赖的聚合权重（对层历史的注意力）
        self.query_proj = nn.Linear(d_model, n_layers)
    
    def forward(
        self,
        current: torch.Tensor,
        history: list[torch.Tensor],
    ) -> torch.Tensor:
        """
        current: (B, D) — 当前层表示
        history: list of (B, D) — 所有先前层的表示
        
        计算对历史的注意力加权求和。
        """
        if not history:
            return current
        
        # 计算内容依赖的权重
        raw_weights = self.query_proj(current)[:, :len(history)]  # (B, n_prev)
        alpha = torch.softmax(raw_weights, dim=-1)                  # (B, n_prev)
        
        # 用学习到的权重聚合
        history_stack = torch.stack(history, dim=1)   # (B, n_prev, D)
        aggregated = (alpha.unsqueeze(-1) * history_stack).sum(dim=1)  # (B, D)
        
        return aggregated + current  # 保留残差连接

    def forward_static_weights(
        self,
        current: torch.Tensor,
        history: list[torch.Tensor],
    ) -> torch.Tensor:
        """
        备选方案：均匀权重（非内容依赖）。
        比学习的注意力更简单；仍然打破了深度均匀性（与标准残差相比）
        因为它池化所有先前层，而不仅仅是紧邻的前一层。
        """
        if not history:
            return current
        n_prev = len(history)
        # 对历史的均匀平均 — 这个变体不需要学习
        alpha = torch.full((n_prev,), 1.0 / n_prev, device=current.device)
        history_stack = torch.stack(history, dim=1)   # (B, n_prev, D)
        return (alpha.unsqueeze(0).unsqueeze(-1) * history_stack).sum(dim=1) + current
```

**诊断问题**：对于你架构的每个维度（深度、宽度、空间、时间、批量、模态），问：沿这个维度同等对待所有元素是一个正确的假设吗？如果不是，你已经识别出一个值得移除的架构约束。

## 维度三：可分性约束

**核心问题**：这个架构在哪些维度上独立（分别）处理，你的任务是否要求它们耦合？

可分性是按顺序处理维度（先空间后通道；先时间后模态）而非联合处理的架构选择。当维度确实是独立的时，这是一种有效的计算捷径——当它们不独立时就会失效。

CNN 中最常见的可分性假设是**空间-通道可分性**：卷积首先聚合空间信息（在核内），然后混合通道（跨输出维度）。这对自然图像识别有效，因为纹理和形状信息通常可以独立提取。它对于需要空间-语义纠缠的任务会失败：知道同一个空间模式在一种语境中意味着"翅膀"而在另一种语境中意味着"鳍"，需要联合的空间-语义处理。

```python
class DepthwiseSeparableConv(nn.Module):
    """
    标准可分离卷积：空间和通道维度独立处理。
    高效但假设空间和通道信息是可分的。
    """
    def __init__(self, in_channels: int, out_channels: int, kernel_size: int = 3):
        super().__init__()
        # 空间聚合：每个通道独立
        self.depthwise = nn.Conv2d(in_channels, in_channels, kernel_size, 
                                    padding=kernel_size//2, groups=in_channels)
        # 通道混合：每个空间位置独立
        self.pointwise = nn.Conv2d(in_channels, out_channels, 1)
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # 先处理空间，再处理通道——从不联合
        x = self.depthwise(x)  # 空间：每个通道只能看到自己
        x = self.pointwise(x)  # 通道：每个位置只能看到自己
        return x


class JointSpatialChannelAttention(nn.Module):
    """
    联合空间-通道处理：在位置 (h, w, c) 的 token
    能关注什么由空间邻近性和通道身份共同决定。
    
    更昂贵但打破了可分性假设——模型可以学习
    "在位置 (h, w) 的模式"在不同通道中意味着不同的事物。
    """
    def __init__(self, d_model: int, n_heads: int = 8):
        super().__init__()
        self.spatial_channel_attn = nn.MultiheadAttention(
            embed_dim=d_model, num_heads=n_heads, batch_first=True
        )
        self.norm = nn.LayerNorm(d_model)
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """
        x: (B, H*W*C, D) — 展平的空间+通道 token
        每个 token 是一个 (h, w, c) 三元组，而不是单独的空间位置。
        注意力在联合空间上——位置和通道同时考虑。
        """
        attn_out, _ = self.spatial_channel_attn(x, x, x)
        return self.norm(attn_out + x)
```

**可分性诊断**：对于维度 A 和 B，问：在不知道 B 的情况下，你能根据 B 的值计算 A 的最优函数吗？如果能，A 和 B 对这个任务是可分的。如果不能——如果你在哪个通道内决定了正确的空间聚合——可分性假设正在引入错误。

**深度-时间可分性**是与具有长程依赖的序列建模最相关的约束。标准 transformer 将深度维度（层号）与时间维度（序列位置）完全分开处理。没有机制让"模型在第 3 层对 token 47 做了什么"直接影响"它在第 7 层对 token 12 做什么"。这是否重要取决于你的任务是否需要这种跨深度的时间交互。

## 使用框架：诊断流程

给定一个架构或任务，按顺序应用三个维度：

**第一步：确定所需的交互阶数。** 最优解需要的最小交互阶数是多少？如果你能从成对交互计算输出，二阶就够了。如果不能，你需要高阶机制或深层堆叠（伴随梯度路径成本）。

**第二步：扫描均匀性违反。** 对于每个维度（深度、宽度、空间、时间、批量、模态），问：这个任务是否需要非均匀处理？标记均匀性错误的位置。这些是强制性的架构债务所在——如果你不修复它们，你是在用模型规模来补偿结构性错误。

**第三步：识别必要的耦合。** 对于每对维度，问：维度 A 的最优处理是否需要知道维度 B？如果是且架构分开了它们，那就是可分性错误——架构无法表示的信息。

```python
from dataclasses import dataclass, field

@dataclass
class ArchitecturalAudit:
    """将三维度诊断应用于架构的输出。"""
    
    # 维度一：交互顺序
    max_representable_order: int
    task_required_order: int
    order_gap: int = 0
    
    # 维度二：均匀性
    uniformity_violations: list[str] = field(default_factory=list)
    # 例如：["depth: 任务需要动态层选择",
    #         "spatial: 任务需要非均匀采样密度"]
    
    # 维度三：可分性
    required_couplings: list[tuple[str, str]] = field(default_factory=list)
    # 例如：[("spatial", "channel"), ("depth", "time")]
    
    def __post_init__(self):
        self.order_gap = max(0, self.task_required_order - self.max_representable_order)
    
    def severity(self) -> str:
        """粗略的严重性分类。"""
        issues = len(self.uniformity_violations) + len(self.required_couplings)
        if self.order_gap > 0:
            return "CRITICAL (交互阶数不匹配)"
        if issues > 3:
            return "HIGH (多个均匀性/可分性约束)"
        if issues > 1:
            return "MEDIUM"
        return "LOW"

# 示例：审计标准 Transformer 用于场景理解
transformer_audit = ArchitecturalAudit(
    max_representable_order=2,       # 注意力是成对的
    task_required_order=4,           # 场景 = 4+ 联合定义的对象
    uniformity_violations=[
        "depth: 所有层在残差求和中权重相等",
        "temporal: 无论远近的上下文权重相同，不考虑近因",
    ],
    required_couplings=[
        ("spatial", "semantic"),     # 场景区域具有依赖语境的意义
        ("modality", "temporal"),    # 在视频中，视觉和音频共同变化
    ],
)

print(f"严重性: {transformer_audit.severity()}")
print(f"阶数差距: {transformer_audit.order_gap}")
print(f"均匀性违反: {len(transformer_audit.uniformity_violations)}")
```

## 框架的目的

三个维度框架不告诉你如何构建架构。它告诉你你的架构正在施加什么约束，这样你就可以检查这些约束对你的任务是否正确。

大多数架构论文不陈述它们的隐含假设。它们展示与提出的机制一致基准结果，但基准可能无法区分"机制有效"和"机制增加了参数"或"基准与架构的均匀性假设一致"。

使用这三个维度迫使你问这个问题：这个架构对这个任务是正确的，还是它仅仅是正确架构的最佳可用近似？如果是后者，性能上限由近似误差决定，更多算力最终会碰到这个上限。如果是前者，你正在使用正确的归纳偏置，扩展变得有成效而非补偿性。
