一次 ROME 编辑是永久性提交。你对特定 MLP 层计算一个 rank-one 更新，将其乘入权重矩阵，保存检查点，此后每次前向传播都会看到那个变化。埃菲尔铁塔现在"住"在罗马了。这个编辑在重启后依然持久，可复现、可审计——你可以精确记录哪些权重发生了多大变化。

测试时训练（TTT）提出了一个对立的契约。同样的 rank-one——或低秩——更新是在推理过程中根据**局部**信号计算的，仅作用于当前前向传播或当前会话的范围，之后丢弃。模型的持久权重从未被修改。这种适应存在于一个短暂状态中，看起来非常像 KV 缓存，只不过它更新的是参数而非激活值。

这两种方法——静态模型编辑和动态测试时适应——经常被当作同一问题的竞争解决方案来讨论。它们不是。它们是解决不同问题的方案，恰好用了相似的数学。理解各自在何处有效、何处失效，能告诉你"知识"在 transformer 内部实际上意味着什么。

## Sun 等人的框架与 RNN 连接

将 TTT 作为层原语的最清晰形式化，来自 Sun、Li、Dalal、Xu、Vikram、Zhang、Dubois、Chen、Wang、Koyejo、Hashimoto、Guestrin 的 2024 年论文"Learning to (Learn at Test Time): RNNs with Expressive Hidden States"。该论文的核心操作简单而精确：RNN 的隐藏状态 $h_t$ 和自注意力层的 KV 缓存都是压缩过去上下文的方式，但两者都有固定容量。隐藏状态将所有过去压缩为单个向量。KV 缓存原样保留所有过去。两者都无法让过去上下文的表示在处理更多上下文时**扩展容量**。

TTT 层用一个小神经网络的权重替代隐藏状态。当新 token 到来时，该层运行一步梯度——不是对主模型，而是对这个内部网络——使用从当前输入计算的自监督损失。内部网络的容量是固定的，但其**表达能力**远比一个向量高，因为相同数量的参数可以表示比向量能表示的位置多得多的不同函数。"隐藏状态"变成了一个在前向传播中学习的小模型。

对任何实现过 ROME 的人来说，这种相似性是直接的。ROME 求解一个 rank-one 优化来更新特定 MLP 的 $W_{\text{down}}$，使得 $W_{\text{down}} k^* = v^*$，其中 $k^*$ 是主体 token 的投影键，$v^*$ 是期望的输出向量。TTT 使用自监督损失对一个小参数函数运行 SGD。两者都是"通过直接调整权重来学习从键到值的映射"。区别在于调整的位置和持续时间。

## 同样的数学如何变成不同的事物

看到分歧最有指导意义的方式是列出每种方法**控制**什么、**假设**什么。

**ROME 控制**：编辑哪一层、编辑的幅度、目标 $(k^*, v^*)$ 对。**ROME 假设**：被编辑的事实可以被定位到特定层的单个 MLP；主体 token 可靠地触发该 MLP；编辑可以在不干扰附近事实的情况下完成。这些假设在经验上是脆弱的。局部性假设是第一个失效的——分布在多层或与其他事实在叠加态中存储的事实无法被干净地编辑，编辑对相关事实的"涟漪"是 ROME 编辑模型经常在改写或相邻事实上失败的主要原因。

**TTT 控制**：内部网络的架构、自监督损失、每个 token 的梯度步数。**TTT 假设**：损失定义了一个信号，当被最小化时会提升主模型对当前输入的输出质量；适应可以被限制到足以不破坏通用行为；梯度步的计算成本值得质量提升。这些假设更易处理，但它们将工程负担转移到了损失设计上。一个糟糕的自监督损失会让模型变差。一个设计良好的——例如重建被掩盖的跨度，或基于上下文预测被删除的 token——能提升事实回忆和长程连贯性。

当被推向极限时，两种方法以相反的方式失效。ROME 在编辑过大或链接多个编辑时（MEMIT 的主要贡献是使其在几千次编辑内可控，超过这个数量模型的通用能力会退化）通过破坏下游能力失效。TTT 因速度慢而失效——每次前向传播现在都包含梯度步——且会漂移，因为内部网络在 token $N$ 时的状态依赖于它在 token $0$ 到 $N-1$ 上训练过的所有内容。

## 可解释性不对称

这是微妙的点。ROME 编辑在强意义上是可解释的：你可以读出 diff、层、幅度和目标事实。TTT 层只在弱意义上可解释：你知道损失和优化器，但内部网络权重在会话中的轨迹是一个动力系统，而非你可以 diff 的静态事物。

这对于对齐尤其重要。ROME 式手术的吸引力在于它给你一份审计日志。你可以说："在这个日期，我们编辑了第 19 层的 MLP 以移除 X 和 Y 之间的关联，使用了这个特定的 rank-one 更新。"如果监管者问为什么你的模型不再产生某些有害输出，你指向这次提交。你还可以反事实推理：如果没有这个编辑，模型会做什么？恢复 diff 并找出答案。

TTT 破坏了这个审计叙事。没有提交。产生有害输出时（或没有产生时）的内部网络是特定梯度轨迹的产物，取决于用户的会话。你无法轻易撤销编辑，因为没有编辑可撤销——只有连续漂移，每次运行产生不同的权重。你可以保存内部状态的检查点，但那只是保留了轨迹的一个时刻，而非保留了原则性决策。

如果目标是每个会话的能力——适应特定文档、方言风格、领域词汇或当前任务——TTT 有明显优势，前提是损失设计良好且会话有足够信号。如果目标是持久对齐——保证模型在部署的操作生命周期内，在 Y 输入分布下不会产生 X 输出——TTT 引入了一种静态编辑没有的连续漂移问题，净比较取决于哪种特性主导生产风险模型。

## 一个具体例子：上下文中的事实 vs. 编辑过的事实 vs. TTT 适应的事实

考虑三种方法告诉模型"截至 2026 年，Acme Corp 的 CEO 是 Jane Smith"：

**在上下文中。** 你把"Acme Corp CEO: Jane Smith"放在系统提示或 RAG 结果中。模型将其视为输入。行为：模型会在输入保持在上下文中时使用这个事实，但后续输入可以轻易覆盖它，模型没有持久信念。延迟成本：极小。审计：极佳（事实原文在提示中）。

**ROME 编辑。** 你计算对 MLP 第 19 层的 rank-one 更新，使"Acme Corp"的键向量映射到编码"Jane Smith"的值向量。行为：模型跨会话使用这个事实，经得住上下文重置，迁移到改写后的查询（"谁运营 Acme？"），但可能在相邻事实上产生不一致行为（"Jane Smith 什么时候出生的？"可能被破坏）。延迟成本：编辑提交后为零。审计：强——你有精确的权重 diff。

**TTT 适应。** 你在推理过程中使用自监督损失微调内部网络的权重，该损失包含对 Acme Corp 相关被掩盖文本的重建。行为：模型在适应后的会话中使用这个事实，一致性取决于自监督信号有多强；会话结束时事实消退。延迟成本：显著——每个 token 都要梯度步，即使内部网络很小。审计：弱——"编辑"是穿过权重空间的轨迹，而非 diff。

这三种不是在一个单一质量轴上竞争的选项。它们是不同的契约。上下文是高审计、低持久。ROME 是高审计、高持久、低灵活性。TTT 是低审计、中等持久、高灵活性。

## 改变一切的算力曲线

TTT 被认真对待的原因——尽管有可审计性问题——是推理算力现在每 FLOP 比 2022 年 ROME 发表时更便宜了，而 agent 工作流的前沿现在发生在推理时。一个自主编码 agent 在任务上运行一小时，在单个会话上花费了服务节点预算的相当一部分。在每个 token 上对小型内部网络添加一步梯度，其边际成本相对于主模型前向传播的成本很小，特别是对于 100B+ 参数的前沿模型——内部网络的参数比基础模型少几个数量级，所以其 SGD 步被主前向成本主导，而非自身主导。

这种算力不对称颠覆了旧有的模型编辑 vs. 预训练的权衡。旧权衡：预训练耗费数百万 GPU 小时，编辑只需秒级。Agent 工作流的新权衡：向共享模型提交编辑会影响所有用户且回滚缓慢；为特定 agent 会话运行 TTT 只影响该会话，会话结束时自动回滚。

换句话说：当编辑应该全局可见且持久时，ROME 是正确答案。当编辑应该局部范围且自动过期时，TTT 是正确答案。一个面向许多用户部署模型的生产系统可能两者都需要——用于策略级事实的 MEMIT 风格批量编辑（"不要讨论 X"），以及用于领域适应的每个会话 TTT（"这个用户在肿瘤学领域工作，调整词汇和先验"）。

## 尚未建成的参考架构

这个技术栈成熟部署的样子已经足够清晰可以勾勒。基础模型是前沿规模，使用标准方法训练，按季度更新。MEMIT 编辑处理策略级知识管理——移除已弃用的事实、插入更正、清除有害关联——并以类似源代码的方式进行版本控制：diff、审查、回滚。TTT 层处理每个会话的适应，受预算策略控制：新用户无适应，受信任上下文每个 token 几步梯度，长时运行的 agent 持续微调其内部网络。

这个架构的治理故事尚未解决。MEMIT 编辑相对容易审计和批准，因为 rank-one 更新列表是可审查的工件。TTT 适应的模型无法审计，因为适应是会话输入的函数。问"这个模型是否被修改以避免产生 X 类内容"的监管者，对 MEMIT 有清晰的答案，对 TTT 有模糊的答案："模型已被训练来抵抗 X，但会话范围的内部网络状态可能根据会话的输入历史保留或丧失那种抵抗力。"

这是即将到来的可解释性账单。该领域花了五年时间使模型编辑可审计。测试时训练——在推理时而非训练时部署相同的数学操作——重新打开了编辑曾开始关闭的每一个问题。

## 值得关注的事项

2026 年值得追踪的具体技术标志：

**使 TTT 可 diff 的混合层。** TTT 层的轨迹可以分解为可解释的组成部分——"这一步是在学习词汇""这一步是在调整主题 X 的先验"——将恢复部分审计叙事。在 TTT 内部状态上使用稀疏自编码器的早期工作是显而易见的方向。

**会话范围策略执行。** 部署模式：在 TTT 适应的主模型前放置一个非适应的"护栏"模型，将任何分歧视为需要二次审视的理由。这是 TTT 版的宪法过滤器。

**适应诱导的越狱。** 显而易见的攻击：设计一个用户会话，其输入导致 TTT 内部网络漂向产生基础模型会拒绝的输出的状态。因为漂移由自监督损失而非策略损失驱动，这种攻击向量不同于提示注入，潜在更持久。

**分离质量与可审计性的基准。** 到目前为止，TTT 在下游任务准确性上评估。下一代基准将分别评分"模型是否正确回答"与"我们能否重建原因"。那将是 ROME vs. TTT 比较的解决之处。
