## 核心问题

深度学习架构里有一个几乎从未被正式质疑过的隐藏公设：**网络的"深度"是一个由整数层号组成的有序链条**。

从 2015 年的 ResNet 到 2026 年的 Attention Residuals，几乎所有主流工作都在这条链上做文章——有的优化残差加法的权重，有的改变跳连的拓扑，有的用注意力替换固定加法——但没有人真正问：**"深度"能不能是一个可寻址的几何对象，而不是一个整数编号？**

这是理解过去十年所有"深度/残差"类工作的核心分水岭。

---

## 一、两种范式的根本分野

### 范式 A：修补残差（Residual Tuning）

这类工作的共同假设是：

- 网络由**离散的操作块**堆叠而成
- 优化目标是改善这些块之间的**信息流**
- "层"作为基本单元的必要性从未被质疑

**代表工作**：

| 工作 | 核心改动 | 本质 |
|------|----------|------|
| Hyper-Connections | 拓宽残差流，允许多流动态组合 | 拓扑级修补 |
| mHC (DeepSeek) | Sinkhorn 投影至双随机矩阵流形 | 守恒约束修补 |
| DeepCrossAttention | 基于输入内容的动态层间权重 | 权重级修补 |
| Attention Residuals | Softmax 注意力替代加法累积 | 聚合算子修补 |
| DenseFormer | 深度加权平均（DWA） | 权重级修补 |

这类工作有真实的工程价值，也解决了"隐藏状态幅度无控增长"和"早期信息稀释"等实际问题。但它们在**拓扑学意义上**从未离开"离散层"这个概念。

### 范式 B：重写深度本体（Rewriting Depth Ontology）

这类工作试图推翻"层"作为离散处理单元的必要性，把"深度"重新定义为其他东西：

| 工作 | 把深度重写为… |
|------|---------------|
| Neural ODE | 连续时间 ODE 的解轨迹 |
| DEQ | 非线性算子的隐式不动点 |
| NDM (2025) | 可微黎曼流形上的局部坐标图 |
| DyMixOp | 积分算子（惯性流形理论） |
| IHGNN | 高阶超图的单调算子平衡态 |

这类工作在数学上更激进，但工程成本通常也更高，且落地难度大。

### 关键洞察

> 当前约 90% 的顶会工作属于范式 A，约 10% 属于范式 B。而**真正的研究机会**在于：范式 A 已把工具推向极限（AttnRes 把"深度注意力化"做到了 drop-in 可用），下一步的价值空间在于：**把 depth 从集合索引升级为连续/相位/几何坐标系，并用可验证机制证明这不是 reparameterization trick**。

## 二、五大默认公理的系统批判

两份报告分别用"三维质疑法"和"第一性原理"框架，独立总结出了几乎相同的隐藏公理体系：

### 公理一：depth 是离散整数层号（有序、可数、链式）

**当前现状**：ResNet 将深度组织为逐层堆叠；Transformer 以堆叠 block 形成深度，残差连接维持训练稳定。

**三维质疑法映射**：这是典型的 **二阶/成对 + 链式** 关系结构，每层主要与近邻发生结构性耦合。

**已被松动的证据**：
- Neural ODE：把离散层视为 ODE 的欧拉离散，用求解器实现"连续深度"
- DEQ：直接求无限深网络的不动点，"深度"消解为平衡条件

**仍未被充分填补的空白**：把深度变成**可插值/可相位对齐的连续坐标**，而不只是"连续化求解"。

---

### 公理二：跨深度信息聚合是"均匀"的（单位权重累加）

**当前现状**：标准 PreNorm 残差形式本质上沿 depth 方向**等权累积**各层输出。Attention Residuals 把这点精确命名为"固定单位权重累积"，指出它导致隐藏态随深度增长、稀释单层贡献。

**三维质疑法映射**：**深度维度均匀性暴政**——默认每层"平等一票"。

**已被松动的证据**：
- DenseFormer：深度加权平均（DWA），可学习历史层复用权重
- DeepCrossAttention：输入相关的权重组合历史层
- AttnRes：Softmax attention over preceding layers

**仍未被充分填补的空白**：这些工作的 depth key 仍隐式等同于"层号"，没有把 depth 变成**连续坐标/相位坐标**，也没有解决"内容—深度纠缠"这一结构性问题。

---

### 公理三：层间更新是局部一步（Markov 式），不是全深度可检索的场

**当前现状**：ResNet/Transformer 的标准形态都把深度当作"逐层推进"的离散动力系统：信息要么被逐层变换，要么通过 shortcut 近似"跳过变换"。

**三维质疑法映射**：关系阶数偏低（局部链式），缺少"集合式、全局式"的深度交互。

**已被松动的证据**：
- Mamba 的选择性 SSM：内容依赖的遗忘/传播，可迁移到 depth 传播
- DenseFormer/AttnRes：全历史层可见，突破 Markov 性

**仍未被充分填补的空白**：把深度变为**可积分的全局算子场**，而不只是全历史注意力。

---

### 公理四：算力/步数在 depth 与 token 上近似均匀分配

**当前现状**：标准 Transformer 对所有 token 运行同样的层数。MoD 把这点作为问题陈述：模型可以学习对不同位置/不同深度动态分配计算。

**三维质疑法映射**：**均匀性假设**在"计算预算维度"的直接体现。

**已被松动的证据**：
- ACT（Adaptive Computation Time）：模型学习需要多少步
- Universal Transformer：权重共享 + 动态停机
- Mixture-of-Depths：token 级 top-k 路由

---

### 公理五：序列维度与深度维度的信息交互完全分离

**当前现状**：标准设计中序列 attention 和 depth 残差是完全独立的子系统。

**最新被质疑**：
- **MoDA（ByteDance, 2026）**：统一序列与深度维度，Query 同时对序列 KV 和深度 KV 做混合 Softmax 检索——这是目前最激进的"维度分离性"松动

## 三、判断框架：如何评价一项"深度/残差"工作

基于上述分析，提出一个可操作的四问框架：

```
Q1. 它质疑了哪条默认公理？
    → 若全部保留，是优化 trick，不是范式创新

Q2. 它的 depth key 是什么？
    → 层号 index：仍是离散层
    → 连续坐标/相位：开始接近 depth 本体

Q3. 它能证明"远层确实被用过"吗？
    → 用 near_vs_far_read_mass 指标验证
    → 若全部塌成近邻，是伪远程检索

Q4. 它的提升能排除"只是更多参数/更稳定优化"的解释吗？
    → 必须做 Iso-FLOPs + Iso-Parameter 对照
    → 必须排除归一化/初始化的混淆变量
```

## 四、结构性结论

从 2015 到 2026 的主线轨迹：

```
梯度高速公路（ResNet/Highway）
    → 深度聚合算子可学习（DenseFormer/DCA）
        → 深度聚合注意力化（AttnRes/MoDA）
            → ？深度作为可寻址几何对象
```

**关键空白**：第三步到第四步之间，没有人真正完成过。大部分方法仍把 depth 当"离散层集合"，而不是"可寻址几何对象（坐标/相位/流形/守恒场）"。

AttnRes 等工作把"深度注意力化"做到工程可用之后，下一步最有价值的空间是：**把 depth 从集合索引升级为连续/相位/对象化的坐标系，并证明这不是 reparameterization trick。**
