## 为什么必须先做红队

“深度本体论”类选题的常见死法不是训练跑不通，而是：

1. 结果可复现，但被审稿人判定为“换皮改写”
2. 结果有提升，但无法排除“多参数/多算力/调参更狠”
3. 理论叙事宏大，但 MVE 证据链不闭合

所以红队审计要先于大规模训练。先证明“不是伪创新”，再谈“是否 SOTA”。

---

## 一、最容易伪创新的 5 种模式

### 模式 1：Attention-Over-Everything

**症状**：把序列 attention 再套一层 depth attention，就宣称范式重构。  
**常见批评**：只是“算子复用”，不是新公理。  
**反制要求**：必须给出“为什么原始残差必然失败”的机制证据，例如 hidden/grad 随深度的结构性退化图，而非只给最终分数。

### 模式 2：几何术语包装（Geometry Washing）

**症状**：论文写“流形/辛结构/守恒”，代码里实质是 LayerNorm + 缩放。  
**常见批评**：几何约束并未在参数化层面成立。  
**反制要求**：给出可验证约束（正交/酉/双随机）与数值验证，不接受口头声明。

### 模式 3：连续化但本质是求解器花活

**症状**：把 ODE/DEQ 的求解技巧当成核心创新。  
**常见批评**：贡献来自 solver，而非 depth 建模对象。  
**反制要求**：分离“求解器收益”和“深度表示收益”，用相同 solver 对比不同深度对象建模。

### 模式 4：只稳不强（Stability-Only）

**症状**：梯度更稳、loss 更平滑，但远层读取和长程能力无提升。  
**常见批评**：仅是优化正则化技巧。  
**反制要求**：必须附带读写能力指标（near/far read mass、深度读熵、远层召回）。

### 模式 5：复杂度升级掩盖真实收益

**症状**：性能涨幅伴随参数/FLOPs/Wallclock 明显上升。  
**常见批评**：多算力换分数。  
**反制要求**：严格执行 matched params + matched FLOPs + matched wallclock。

## 二、最容易被 Rebuttal 击穿的攻击面

| 攻击面 | 典型评审问题 | 你必须提前准备的证据 |
|--------|-------------|----------------------|
| 参数不公平 | “同参数量还成立吗？” | Iso-Parameter 对照 |
| 算力不公平 | “同 FLOPs 还成立吗？” | Iso-FLOPs 对照 |
| 速度不公平 | “同 wallclock 还成立吗？” | Iso-Wallclock 对照 |
| 稳定化混淆 | “是不是只是换了归一化/初始化？” | 控制变量实验（固定 LN/init） |
| 近层塌缩 | “远层真的被读取了吗？” | near_vs_far_read_mass + depth_read_entropy |
| 重参数化嫌疑 | “是不是等价于 layer embedding？” | 功能等价剥离实验 |

## 三、强制实验协议（不满足就不投稿）

### 1) 三个公平性对照

1. **Iso-Parameter**：同参数量  
2. **Iso-FLOPs**：同计算预算  
3. **Iso-Wallclock**：同训练/推理时间预算

任何一个缺失，都会被认为贡献归因不完整。

### 2) 三组深度几何诊断

1. **传播稳定性**：`hidden_rms_by_depth`、`grad_rms_by_depth`  
2. **读取行为**：`near_vs_far_read_mass`、`depth_read_entropy`  
3. **结构一致性**：`phase_alignment_score` 或 `slot_specialization`

### 3) 三类负对照

1. **随机深度键**：替换真实深度键，验证是否仍同样有效  
2. **去结构约束**：移除正交/流形/相位约束，测性能回落幅度  
3. **简化基线增强**：给基线同等参数和计算，排除“只是更大模型”

## 四、MVE 优先级与止损纪律

### 优先级建议

1. **优先跑低成本强证伪方向**：相位寻址、slot 化、诊断协议  
2. **再跑中成本机制方向**：守恒输运、预算化深度记忆  
3. **最后跑高成本范式方向**：积分核、隐式超图、相位 DEQ

### 统一止损线

| 条件 | 动作 |
|------|------|
| 连续 3 次实验仅提升 < 0.5% 且方差大 | 暂停该方向 |
| 无法通过 Iso-FLOPs/Parameter 对照 | 不进入下一阶段 |
| 读写能力指标无显著改善 | 判定为“只稳不强” |
| 推理延迟提升 > 2x 且收益不足 | 转为工程备选，不走论文主线 |

## 五、投稿前红队清单（逐条打勾）

- [ ] 是否能一句话说清“被推翻的默认公理”  
- [ ] 是否有明确的最近邻工作与结构差异，而非概念差异  
- [ ] 是否具备 Iso-Parameter / Iso-FLOPs / Iso-Wallclock 三对照  
- [ ] 是否给出深度读取行为证据（不是只有最终分数）  
- [ ] 是否有负对照证明“去掉关键机制就失效”  
- [ ] 是否有失败案例分析（不仅报告成功样本）  
- [ ] 是否存在明确 stop-loss 规则并被实际执行

若有任意 2 项无法勾选，建议先补实验，不要着急投稿。

---

## 六、30 天执行模板（可直接套用）

### 第 1 周：定义与复现

1. 固定基线、数据集与预算协议  
2. 复现 1 个最近邻工作（例如 AttnRes 或 DCA）  
3. 接入诊断工具链

### 第 2 周：MVE 与负对照

1. 跑主机制 MVE  
2. 同时跑随机键/去约束负对照  
3. 判定是否跨过最小可发表门槛

### 第 3 周：公平性对照

1. 补齐 Iso-Parameter / FLOPs / Wallclock  
2. 完成误差条与统计显著性  
3. 产出可复现实验脚本

### 第 4 周：红队复盘与稿件成型

1. 按红队清单逐条审计  
2. 提前撰写 rebuttal 草案（按攻击面逐项回应）  
3. 只保留能被证据链支撑的主张

---

## 结论

Part 5 的核心不是“更保守”，而是“更高胜率”。  
对于深度本体论这种高概念方向，**先做红队，后做扩训**，是避免投入失真和投稿翻车的最低成本路径。
