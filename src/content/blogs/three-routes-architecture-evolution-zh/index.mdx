残差连接是深度学习中最少被质疑的设计决策之一。自2015年ResNet使训练152层网络变得可行以来，`h_{l+1} = h_l + f(h_l)` 这一模式变得如此普遍，以至于它被内置到默认的Transformer实现、PyTorch示例代码和深度学习入门课程中。

但通用不等于最优。残差连接是一个训练稳定性技巧，代价是架构上的成本：它强制所有层均匀聚合，每一层的输出对后续每一层的贡献权重固定为1。三条研究方向从不同角度攻击这一假设，理解它们之间的关系可以澄清下一代架构的发展方向。

## 固定残差的问题

在审视替代方案之前，明确残差连接实际做错了什么会有所帮助。

标准的PreNorm残差堆栈产生：

```
h_L = h_0 + Σ_{l=0}^{L-1} f_l(LayerNorm(h_l))
```

和中的每一项权重都是1。一个学习了有用特征转换的层与一个学了噪声的层对输出的贡献相等。深度网络通过基于梯度的优化在一定程度上缓解了这个问题——贡献为空的层倾向于学习接近零的转换——但结构约束意味着模型无法明确地绕过无用的层或放大特别有用的层。

第二个问题是 `h_{l+1}` 只能看到 `h_l`。即使有跳连扩展到更早的层（如DenseNet），聚合也是硬编码的：每个先前的层都有贡献，但没有基于内容的selection模型无法问"哪个层拥有对当前处理最有相关性的信息？"

这两个限制——固定聚合权重和近邻盲连接——正是三条路径各自要解决的问题。

## 路径A：动态聚合（挑战固定权重）

Moonshot AI的注意力残差论文（arXiv:2603.15031）进行了最直接的攻击：用Softmax加权注意力替代固定权重求和。

不是：

```python
h_next = h_current + f(layer_norm(h_current))
```

模型计算：

```python
# 标准注意力残差
scores = [query(h_current) @ key(h_i) for h_i in history]
weights = softmax(scores)
h_next = sum(w * h_i for w, h_i in zip(weights, history))
```

这给了模型之前缺乏的东西：在每一层，基于这些输出的内容，决定分配给每个先前层输出的权重的能力。一个处理句法结构的层可以增加权重给捕获了表层模式的前面层，同时降低关注语义内容的更深层的权重。

实际的挑战是层数的二次方内存开销。块注意力残差变体通过将层分组并在块内而非全局应用动态聚合来解决这个问题：

```python
class BlockAttnResidual(nn.Module):
    def __init__(self, hidden_dim: int, block_size: int = 4):
        super().__init__()
        self.block_size = block_size
        self.q_proj = nn.Linear(hidden_dim, hidden_dim)
        self.k_proj = nn.Linear(hidden_dim, hidden_dim)
        self.v_proj = nn.Linear(hidden_dim, hidden_dim)
        self.out_proj = nn.Linear(hidden_dim, hidden_dim)
        self.scale = nn.Parameter(torch.ones(1) * 0.1)

    def forward(
        self,
        current: torch.Tensor,
        block_history: list[torch.Tensor]
    ) -> torch.Tensor:
        if not block_history:
            return current

        q = self.q_proj(current)   # [B, T, D]
        history = torch.stack(block_history, dim=2)  # [B, T, L, D]
        k = self.k_proj(history)
        v = self.v_proj(history)

        # [B, T, L]
        scores = torch.einsum("btd,btld->btl", q, k) / (q.shape[-1] ** 0.5)
        weights = scores.softmax(dim=-1)

        # [B, T, D]
        aggregated = torch.einsum("btl,btld->btd", weights, v)
        return current + self.scale * self.out_proj(aggregated)
```

`scale` 参数——初始化为0.1——对训练稳定性至关重要。它确保在初始化时，注意力残差贡献一个小的扰动而非大的信号，允许标准残差动态在注意力权重变得有意义之前主导早期训练。

在Kimi Linear（总计48B / 活跃3B）上的实验显示，块大小为4时，MMLU-Pro上+1.6%，RULER 128K上+2.2%。收益随模型规模放大，表明更大的模型从选择性路由信息的能力中获益更多。

**核心洞察**：路径A回答"哪个先前层现在最相关？"它改变了聚合机制，同时保持层结构完整。

## 路径B：基于查询的编排（改变被聚合的内容）

路径A仍然聚合相同的层。路径B问一个更根本的问题：为什么模型应该聚合层？如果模型可以查询已处理内容的压缩记忆，而不是所有先前的层输出，那会怎样？

这就是编排或基于查询的方法，以 **BLIP-2的Q-Former** 和 **Perceiver IO** 为代表。

Q-Former引入了一组固定的learnable查询向量，通过交叉注意力"采访"大型编码器：

```python
class QFormer(nn.Module):
    def __init__(self, d_model: int, num_queries: int = 32):
        super().__init__()
        # 可学习的查询向量 — 固定数量，不依赖于输入长度
        self.queries = nn.Parameter(torch.randn(num_queries, d_model))
        self.cross_attn = nn.MultiheadAttention(d_model, num_heads=8, batch_first=True)

    def forward(self, encoder_output: torch.Tensor) -> torch.Tensor:
        """
        encoder_output: [B, seq_len, d_model]  (可能是数千个token)
        returns: [B, num_queries, d_model]       (固定的32个token)
        """
        B = encoder_output.shape[0]
        queries = self.queries.unsqueeze(0).expand(B, -1, -1)

        # 可学习的查询从编码器中选择它们需要的内容
        condensed, _ = self.cross_attn(
            query=queries,
            key=encoder_output,
            value=encoder_output
        )
        return condensed  # 固定大小的瓶颈
```

与路径A的关键区别：被查询的"历史"不是层输出的堆栈，而是压缩的潜记忆。查询向量*学习要问什么*，而不是被告知有什么可用的。

Perceiver IO通过持久的潜数组进一步扩展了这一点，该数组通过交叉注意力迭代更新，然后通过输出特定查询有选择地读取：

```python
class PerceiverIOBlock(nn.Module):
    def __init__(self, d_latent: int, d_input: int, num_latents: int):
        super().__init__()
        self.latents = nn.Parameter(torch.randn(num_latents, d_latent))
        # 交叉注意力：潜数组查询输入
        self.cross_attn = nn.MultiheadAttention(d_latent, num_heads=4, batch_first=True)
        # 自注意力：潜数组之间处理
        self.self_attn = nn.MultiheadAttention(d_latent, num_heads=4, batch_first=True)

    def forward(self, inputs: torch.Tensor) -> torch.Tensor:
        B = inputs.shape[0]
        latents = self.latents.unsqueeze(0).expand(B, -1, -1)

        # 潜数组有选择地关注它们从输入中需要的任何内容
        latents, _ = self.cross_attn(latents, inputs, inputs)
        # 潜数组处理所选信息
        latents, _ = self.self_attn(latents, latents, latents)
        return latents
```

架构含义：模型的内部"工作记忆"（潜数组）与输入表示解耦。这使得用固定内存处理任意长的输入成为可能——通过学习将重要的内容压缩成固定大小的潜空间。

**核心洞察**：路径B回答"我实际需要什么信息？"而不是"我应该如何权衡已经处理过的内容？"它设计成本更高（必须决定分配多少潜槽），但在表示能力上更灵活。

## 路径C：连续潜变量（改变训练哲学）

路径A和B都作用于离散层。路径C问层是否本身就是正确的抽象。

VAE框架将对输入到输出的映射视为穿越连续潜空间的路径：

```
x → encoder → μ, σ → z ~ N(μ, σ²) → decoder → x̂
```

重参数化技巧使其可训练：

```python
def encode(self, x: torch.Tensor) -> tuple[torch.Tensor, torch.Tensor]:
    h = self.encoder(x)
    mu = self.mu_proj(h)
    log_var = self.log_var_proj(h)
    return mu, log_var

def reparameterize(self, mu: torch.Tensor, log_var: torch.Tensor) -> torch.Tensor:
    # 训练时：z = mu + epsilon * sigma (可微分！)
    # 推理时：z = mu (确定性)
    if self.training:
        std = (0.5 * log_var).exp()
        epsilon = torch.randn_like(std)
        return mu + epsilon * std
    return mu

def forward(self, x: torch.Tensor):
    mu, log_var = self.encode(x)
    z = self.reparameterize(mu, log_var)
    x_hat = self.decoder(z)

    # 重构损失 + KL散度
    recon_loss = F.mse_loss(x_hat, x)
    kl_loss = -0.5 * (1 + log_var - mu.pow(2) - log_var.exp()).sum(dim=-1).mean()
    return x_hat, recon_loss + self.beta * kl_loss
```

`beta` 参数（来自β-VAE）控制重构质量和潜空间结构之间的权衡。高β迫使模型使用正交的、可解释的潜维度——对想要操作特定属性的下游任务有用。低β优先考虑重构质量。

与路径A和B的联系：VAE框架将*深度*视为潜空间中的连续坐标，而不是离散的层索引。神经ODE通过用连续微分方程替换离散层递归使这一点变得明确：

```
dh(t)/dt = f(h(t), t, θ)
```

这是路径C方向的理论终点。模型没有层；它有一个由学习向量场参数化的连续潜空间轨迹。在实践中，ODE求解器使训练变得缓慢和内存密集，这就是为什么当前工作（如早期帖子中概述的CDMN框架）专注于保持连续深度语义而不需要ODE求解器的离散近似。

**核心洞察**：路径C问"信息空间的正确结构是什么？"而不是"我应该如何聚合或查询当前结构中的内容？"它是技术上最激进但也最昂贵的。

## 三条路径的关系

三条路径不是同一个问题的竞争答案。它们在攻击不同的结构假设：

| 路径 | 被挑战的假设 | 机制 | 成本 |
|------|------------|------|------|
| A: 动态聚合 | 所有层贡献相等 | 学习对层历史的注意力权重 | O(L²) 内存 |
| B: 基于查询 | 聚合所有先前状态 | 压缩潜记忆 + 基于内容的查询 | 固定潜槽 |
| C: 连续潜 | 离散层是正确的抽象 | 连续深度坐标 | ODE求解器/近似 |

它们也可组合。注意力残差（路径A）可以应用于Perceiver风格的潜（路径B）内部，使潜更新更有选择性。连续深度思想（路径C）可以使用应用于块注意力架构（路径A）的相位编码（RDE）进行离散近似。

对于今天构建系统的人的实际进展：

1. **从路径A开始（块注意力残差）**如果你有标准Transformer并想要通过包含的工程变更获得可衡量的收益。基于已发布的消融，块大小4给出了最佳计算/准确度权衡。

2. **当你的输入长且异构时，添加路径B思维（潜压缩）**。Q-Former模式特别干净：学习要问什么，而不是存储什么。

3. **将路径C保持在视野中**。连续深度框架今天还不能以理想形式部署，但概念转变——深度作为内存地址而非处理阶段——在设计路径A和B时影响你的思考方式。

## 剩余的难题

三条路径都使层聚合更智能。它们都没有完全解决我之前帖子中所说的"一致性暴政"：相同的聚合策略应该均匀应用于所有输入、所有位置和所有深度坐标的假设。

路径A改变聚合权重但在各处应用相同的注意力机制。路径B使用固定潜结构不管输入内容。路径C为整个潜空间指定单个向量场。

下一个前沿是*元学习*聚合：一个模型不仅学习要聚合什么，还学习*聚合策略应该如何根据输入复杂性、任务类型或计算中的位置而变化*。这更接近神经科学图景——不同皮层区域使用根本不同的整合策略，而不是相同机制不同权重。
