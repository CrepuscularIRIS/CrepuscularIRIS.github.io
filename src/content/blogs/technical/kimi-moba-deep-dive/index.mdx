## 概述

MoBA (Mixture of Block Attention) 是 Moonshot AI (Kimi) 团队于 2025年2月18日发布的研究成果，将 Mixture of Experts (MoE) 原理应用于注意力机制，实现长上下文 LLM 的高效处理。

## 论文信息

- **标题**: MoBA: Mixture of Block Attention for Long-Context LLMs
- **arXiv**: [2502.13189](https://arxiv.org/abs/2502.13189)
- **发布日期**: 2025-02-18
- **作者**: Enzhe Lu, Zhejun Jiang 等 23 人（Moonshot AI）
- **GitHub**: [MoonshotAI/MoBA](https://github.com/MoonshotAI/MoBA)
- **DOI**: 10.48550/arXiv.2502.13189

## 核心贡献

### 1. MoE 原理应用于注意力机制

首次将 Mixture of Experts (MoE) 的路由机制应用于注意力层，实现：
- 全上下文分块处理
- 选择性注意力聚焦

### 2. 无参数 Top-K 门控

创新的无参数门控机制：
- 每个查询 token 学习关注最相关的 KV 块
- 动态选择 top-k 相关块
- 无需额外训练参数

### 3. 全/稀疏注意力无缝切换

支持在两种模式间灵活切换：
- **全注意力** - 传统 Transformer 模式
- **稀疏注意力** - MoBA 高效模式

### 4. 生产环境部署

已在 Kimi 生产环境实际部署，验证了其工业级可用性。

## 技术架构

```
输入序列 → 分块 → 查询路由 → 选择 Top-K KV 块 → 局部注意力 → 聚合输出
```

### 关键创新

| 特性 | 描述 |
|------|------|
| 分块策略 | 全上下文均匀分块 |
| 路由机制 | 无参数 top-k 选择 |
| 注意力计算 | 仅在选中的块上进行 |
| 模式切换 | 全注意力和稀疏注意力无缝切换 |

## 开源实现

GitHub 提供两种后端实现：

### 1. moba_naive

基于注意力掩码的实现：
- 易于理解
- 适合研究和学习

### 2. moba_efficient

生产优化实现：
- 最高 40 倍加速（32K 序列、1 头、2048 块、topk=3）
- 依赖: flash-attn==2.6.3, torch>=2.1.0

### 性能基准

| 配置 | 加速比 |
|------|--------|
| 32K 序列、1 头、2048 块、topk=3 | 最高 40x |
