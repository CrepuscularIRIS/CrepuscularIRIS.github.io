## 实验目标

验证三个核心假设：
1. **AttnRes 优于经典 Residual**：动态权重是否解决 hidden state 膨胀
2. **RDE 可行**：相位编码的 depth retrieval 是否有稳定收益
3. **连续化方向正确**：从 Block 到连续的演进是否值得追求

---

## 实验 1：三种 Residual 变体对比

### 实验设置

在同构 Transformer 上实现三个版本：

```python
# 版本 A: 经典 Residual (Baseline)
class ClassicResidual(nn.Module):
    def forward(self, x):
        return x + self.f(x)

# 版本 B: Attention Residuals (AttnRes)
class AttentionResidual(nn.Module):
    def __init__(self, lookback_window=4):
        self.attention = MultiHeadAttention()
        
    def forward(self, x, history):
        # history = [h_{t-4}, ..., h_{t-1}]
        query = x
        keys = values = torch.stack(history)
        attn_out, _ = self.attention(query, keys, values)
        return x + attn_out

# 版本 C: Query-over-depth (RDE 简化版)
class QueryOverDepth(nn.Module):
    def __init__(self, num_landmarks=8):
        self.landmark_states = nn.Parameter(torch.randn(num_landmarks, d_model))
        self.query_proj = nn.Linear(d_model, d_model)
        
    def forward(self, x, depth):
        query = self.query_proj(x)
        # 简化版：直接 attention 到 landmark states
        attn_out, _ = self.attention(query, self.landmark_states, self.landmark_states)
        return x + 0.1 * attn_out  # 小残差系数
```

### 观察指标

| 指标 | 测量方法 | 预期结果 |
|------|----------|----------|
| **Hidden state norm** | 每层输出 L2 norm | AttnRes/RDE 更稳定，不单调膨胀 |
| **梯度分布** | 每层梯度范数 | AttnRes/RDE 更均匀 |
| **长程依赖** | Path-X, Long Range Arena | AttnRes/RDE 优于 Baseline |
| **计算/显存** | 训练时间, GPU memory | AttnRes 增加 20-30%, RDE 增加 10-15% |

### 实验数据集

- **短序列**: CIFAR-10 (图像分类, ~1K tokens)
- **中序列**: Wikitext-103 (语言建模, ~1K-4K tokens)
- **长序列**: Path-X (全局依赖, 16K tokens)

### 成功标准

| 版本 | 成功标准 |
|------|----------|
| AttnRes | 长序列任务提升 &gt;5%, hidden norm 稳定 |
| RDE | 不劣于 AttnRes, 计算增加 &lt;20% |

## 实验 2：RDE 相位寻址验证

### 实验目标

验证"是否存在稳定的相位寻址收益"，**先不追全连续化**。

### 简化版 RDE

```python
class RotaryDepthEmbedding(nn.Module):
    def __init__(self, dim, num_depths=16):
        # 离散深度位置，但用 RoPE 编码
        self.depth_positions = torch.linspace(0, 1, num_depths)
        self.inv_freq = 1.0 / (10000 ** (torch.arange(0, dim, 2).float() / dim))
        
    def encode(self, depth_idx):
        # depth_idx: 0-15 的离散索引
        angles = self.depth_positions[depth_idx] * self.inv_freq
        return torch.cos(angles), torch.sin(angles)

class RDEResidual(nn.Module):
    def __init__(self, d_model, num_landmarks=8):
        self.rde = RotaryDepthEmbedding(d_model, num_landmarks)
        self.landmark_states = nn.Parameter(torch.randn(num_landmarks, d_model))
        
    def forward(self, x, current_depth_idx):
        # 查询当前深度
        query = x
        
        # 计算与所有 landmark 的相似度（基于 RoPE 编码）
        cos_d, sin_d = self.rde.encode(current_depth_idx)
        query_rotated = apply_rotary(query, cos_d, sin_d)
        
        # Attention 到 landmark states
        similarities = query_rotated @ self.landmark_states.T
        attn_weights = F.softmax(similarities, dim=-1)
        
        # 加权聚合
        retrieved = attn_weights @ self.landmark_states
        return x + 0.1 * retrieved
```

### 关键验证点

| 验证点 | 方法 | 成功标准 |
|--------|------|----------|
| **相位编码有效** | 比较 RoPE 编码 vs 无编码 | RoPE 版本训练更稳定 |
| **深度检索有效** | 比较固定 landmark vs 动态检索 | 动态检索长序列更好 |
| **相位对齐有意义** | 可视化相似度矩阵 | 相位相近的深度确实检索相关 landmark |

### 简化假设

**先不做全连续化**：
- 深度仍是离散索引 (0-15)
- 但用 RoPE 编码引入"相位"概念
- 验证"相位编码离散化"是否有收益

**如果简化版成功**：
- 再扩展到连续深度 t ∈ [0,1]
- 引入 ODE solver 或近似

## 实验 3：消融实验

### 消融维度

| 维度 | 变体 | 目的 |
|------|------|------|
| **Lookback 窗口** | 2, 4, 8, 16 | 找到最优历史长度 |
| **Landmark 数量** | 4, 8, 16, 32 | 找到最优记忆容量 |
| **残差系数** | 0.05, 0.1, 0.2, 1.0 | 平衡新旧信息 |
| **RoPE 基频** | 10000, 50000, 100000 | 调整相位敏感度 |

### 关键发现

希望通过消融找到：
- 是否存在"甜点"配置
- 哪些超参数对性能影响最大
- 哪些维度可以固定，哪些需要精细调优

## 预期成果

### 成功场景

| 结果 | 结论 | 下一步 |
|------|------|--------|
| AttnRes > Classic | 动态权重有效 | 深入研究 AttnRes 变体 |
| RDE > AttnRes | 相位编码有效 | 扩展到连续深度 |
| RDE ≈ AttnRes | 简化版收益有限 | 重新评估连续化必要性 |
| 都失败 | 假设可能错误 | 回到第一性原理重新审视 |

### 失败信号

如果出现以下情况，需要重新评估方向：
- AttnRes 训练不稳定，难以收敛
- RDE 计算开销过大，速度下降 >50%
- 长序列任务无明显提升
- 消融实验无清晰甜点
