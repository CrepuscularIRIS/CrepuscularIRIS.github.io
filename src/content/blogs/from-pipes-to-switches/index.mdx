The residual network, introduced by He et al. in 2015, solved a real problem: gradient vanishing in very deep networks. It solved it by adding a bypass — a direct connection from layer $l$ to layer $l+1$ that lets gradients flow without passing through the transformation $f_l$.

This bypass is useful. But the framing it created has become a design prison.

The dominant mental model for neural network architecture since ResNet is the **pipeline**: information enters at layer 1, passes sequentially through each subsequent layer, and exits at the final layer as a prediction. Residual connections are "patches" on this pipeline — they ease gradient flow and allow depth, but they preserve the fundamental assumption that processing is sequential and information flows in one direction through a fixed sequence of transformations.

This assumption is architecturally incorrect for most interesting ML tasks, and the field is slowly learning to abandon it. The transition is from pipelines to **switches**: from networks where data flows through fixed sequential stages to networks where information is dynamically routed based on content.

## Why the Pipeline Model Is Wrong

The pipeline assumption says that information entering a network at layer 1 must traverse every subsequent layer before exiting. This is the computational equivalent of forcing every data packet through every router on the network, regardless of where the destination is. It's obviously inefficient for routing problems. It turns out it's also inefficient for deep learning.

Consider what actually happens in a ResNet during inference on a visual input. The first few layers extract oriented edges. The middle layers detect textures and object parts. The deep layers encode semantic categories. When generating a prediction, the final softmax needs category-level information from the deep layers. It doesn't need to re-process the edge information from layer 2 — that information was useful for building texture representations at layer 6, but it's irrelevant by layer 48.

Yet the pipeline forces this. Every intermediate representation accumulates through residual addition:

$$h_{L} = h_0 + \sum_{l=0}^{L-1} f_l(\text{LN}(h_l))$$

The final representation is a linear superposition of every layer's contribution, with equal weight. The important semantic representations from layers 40-48 are diluted by the edge and texture representations from layers 2-20. This is the **PreNorm dilution problem**: the contribution of any given layer to the final representation is $\frac{1}{L}$, regardless of how important that layer's computation actually is.

## The Data Processing Inequality

The information-theoretic argument is clean. For any Markov chain $X \to H_1 \to H_2$, the data processing inequality states:

$$I(X; H_2) \leq I(X; H_1)$$

Information can only be lost, never gained, as you process a signal through intermediate stages. A pipeline is a sequence of Markov steps: information about the input can only decrease with depth.

This isn't necessarily fatal — a well-trained model compresses the input toward a representation that's maximally predictive of the output while discarding task-irrelevant variance. But it has a key implication: the later layers of a pipeline can only work with what the earlier layers preserved. If layer 20 failed to preserve a feature that layer 40 needs, layer 40 cannot recover it.

A **switch network** breaks this constraint. Rather than propagating information through a fixed sequence, a switch network retrieves information from any layer based on the current computation's needs:

$$h_{out} = \sum_{i \in \text{selected}} \alpha_i \cdot h_i$$

where $\alpha_i$ are content-dependent attention weights and the selected set is a sparse subset of all previous layer outputs. The data processing inequality still holds per path — but the switch can query the layer most informative for the current computation, bypassing the information loss that accumulated along other paths.

## The Architectural Evolution

The field has been moving toward switch-like networks for a decade, without always framing it that way.

**DenseNet (2016)** made the connection to all previous layers explicit — each layer receives concatenated outputs from all earlier layers. This is a switch with uniform weights: every previous layer is equally accessible. It works surprisingly well but is computationally intractable at depth because concatenation grows the representation dimension linearly with depth.

**Self-attention in Transformers** is a token-level switch: each position attends to all other positions with content-dependent weights. This is powerful for sequence modeling but doesn't address the depth dimension — each layer still processes the outputs of the immediately preceding layer.

**Mixture of Experts (MoE)** is a width-level switch: for a given layer, only a subset of expert networks is activated per input. This routes computation *within* a layer but doesn't address routing *across* layers.

**Kimi's Block Attention Residuals** (AttnRes, 2026) takes the depth-level switch seriously. Within each block of layers, each layer can attend to all previous layers in the block rather than just the immediately preceding one. The weights are learned, content-dependent, and sparse by construction:

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class BlockAttentionResidual(nn.Module):
    """
    Depth-level switch: each layer attends to all previous layers
    within the current block, rather than only the preceding layer.
    """
    def __init__(self, d_model: int, block_size: int, n_heads: int = 8):
        super().__init__()
        self.block_size = block_size
        self.head_dim = d_model // n_heads
        self.n_heads = n_heads

        self.q_proj = nn.Linear(d_model, d_model)
        self.k_proj = nn.Linear(d_model, d_model)
        self.v_proj = nn.Linear(d_model, d_model)
        self.out_proj = nn.Linear(d_model, d_model)
        self.scale = nn.Parameter(torch.ones(1) * 0.1)

    def forward(self, current: torch.Tensor, history: list[torch.Tensor]) -> torch.Tensor:
        """
        current: (batch, seq, d_model) — current layer representation
        history: list of (batch, seq, d_model) — previous layers in block
        """
        if not history:
            return current

        B, T, D = current.shape
        H = len(history)

        # Current layer queries
        q = self.q_proj(current).view(B, T, self.n_heads, self.head_dim)

        # Stack historical layers as keys/values
        hist_stack = torch.stack(history, dim=2)           # (B, T, H, D)
        k = self.k_proj(hist_stack.view(B * T * H, D)).view(B, T, H, self.n_heads, self.head_dim)
        v = self.v_proj(hist_stack.view(B * T * H, D)).view(B, T, H, self.n_heads, self.head_dim)

        # Attention over depth dimension
        scores = torch.einsum('bthd,btkhd->bthk', q, k) / (self.head_dim ** 0.5)
        weights = F.softmax(scores, dim=-1)                # (B, T, heads, H)

        # Weighted aggregation across history
        aggregated = torch.einsum('bthk,btkhd->bthd', weights, v)
        aggregated = aggregated.view(B, T, D)
        output = self.out_proj(aggregated)

        return current + self.scale * output
```

The difference from a standard residual is in the `history` argument: instead of `h_{l+1} = h_l + f(h_l)` (add immediately preceding layer), this computes attention-weighted aggregation across all layers in the block. The weights are input-dependent — layer 10 might weight layer 3 heavily for some inputs and layer 8 heavily for others.

## The Sparsity Constraint

Full dense connections across all layers would recover the DenseNet problem: memory grows quadratically with depth. The key design choice that makes depth-level switches tractable is **block sparsity**.

Both MoBA (for sequence attention) and AttnRes (for depth attention) use the same trick: partition the layers into blocks of size $B$, and allow dense attention within each block while using standard residuals between blocks. The complexity drops from $O(L^2)$ to $O(L \cdot B)$ where $B \ll L$.

The block size is a hyperparameter trading off expressiveness against cost. Moonshot AI's experiments on Kimi Linear 48B show that block size 4 is the Pareto-optimal choice: it captures most of the accuracy gains from full AttnRes (which uses block size equal to total layers) while adding only ~8% memory overhead.


The non-monotonic relationship (performance peaks at block size 4 then declines) is interesting. Too-large blocks allow attention across layers that are too dissimilar to have informative keys; too-small blocks don't provide enough cross-layer context. Block size 4 appears to match the natural scale of "processing stages" in the model — roughly one stage per block.

## Routing as the New Primitive

The deeper shift here is conceptual. When the pipeline is the fundamental abstraction, the design question is: *how many layers, and what transformation should each apply?* Layer depth is a proxy for representational capacity, and deeper is generally better up to the training stability limit.

When the switch is the fundamental abstraction, the design question changes: *what should be connected to what, and how sparsely?* The graph topology of the routing connections is the primary architectural choice, not the depth. Depth still matters (more nodes in the routing graph = more capacity), but the routing pattern determines which computations can inform which others.

This reframing has implications for architecture search, initialization, and training. In a pipeline, you can initialize each layer's weights independently and train them to learn transformations. In a switch, the routing weights need to be initialized carefully — a switch that routes everything through a single path has collapsed back to a pipeline. Good switch initialization should spread routing weights broadly, then allow them to specialize.

```python
def initialize_switch_weights(model: nn.Module, uniform_init_scale: float = 0.1):
    """
    Initialize routing/query projections with near-uniform attention,
    so training starts with broad routing and specializes over time.
    """
    for name, param in model.named_parameters():
        if 'q_proj' in name or 'k_proj' in name:
            # Small uniform initialization → near-uniform softmax at start
            nn.init.normal_(param, mean=0.0, std=uniform_init_scale)
        elif 'scale' in name:
            # Residual scaling starts small so switch output is a small perturbation
            nn.init.constant_(param, 0.1)
```

## What This Means for Future Architectures

The current generation of switch networks — MoE, AttnRes, MoBA — are all partial switches. They route within a constrained scope: within a layer (MoE), within a block of layers (AttnRes), or within a block of token positions (MoBA). The fully connected switch — where any layer can attend to any other layer, any position can attend to any other position, across any depth — remains computationally intractable with current hardware and training methods.

But the trajectory is clear. The history of deep learning architecture is a history of relaxing constraints: from fixed receptive fields to learned attention, from fixed depth to dynamic depth routing, from fixed width to mixture of experts. Each relaxation came with a cost that was eventually managed through hardware improvements and engineering ingenuity.

The pipeline will give way to the switch, not because pipelines are wrong but because they're a special case of a more general design space that happens to be easy to train. As training methods improve and hardware becomes cheaper, the special-case constraint will be relaxed in the same way that the fixed-receptive-field constraint of early CNNs was relaxed by self-attention.

The architectures that matter in five years will be those that took the routing question seriously, rather than accepting sequential processing as a given.
