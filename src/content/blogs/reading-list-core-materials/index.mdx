## 最小实验清单（待执行）

基于阅读后的验证计划：

### 实验 1：三种 Residual 变体对比
在同构 Transformer 上实现：
1. 经典 Residual: `h_{l+1} = h_l + f(h_l)`
2. AttnRes: `h_{l+1} = Σ α_i · h_i`
3. Query-over-depth: `h(t) = retrieve(depth_memory, query)`

观察指标：
- hidden state norm 随深度是否单调膨胀
- 每层梯度分布是否更均匀
- 长程依赖任务是否更受益
- 计算/显存代价是否可接受

### 实验 2：RDE 验证
- 实现 Rotary Depth Embedding
- 验证 depth retrieval 是否存在稳定收益
- 先不追全连续化，只验证"相位寻址"可行性

## 红队审计：避坑指南

### 1. 不要把 System Card 当技术细节论文
- System Card 主要讲能力边界和安全评估
- 架构细节极少，信息密度低

### 2. 不要把 Kimi AttnRes 误当成"连续深度"
- AttnRes 仍是离散层之间的选择性聚合
- 只是把固定累加换成 learned aggregation
- 不是真正的连续化

### 3. 不要过早在异构 backbone 上验证 RDE
- 先选表示空间尽量同构的 Transformer 变体
- 避免把"表示不兼容"误判成"理论错误"

### 4. 不要把 VAE 和 Orchestration 混成一类结论
- VAE：连续潜变量建模妥协
- Q-Former / Perceiver：查询式编排与读写机制
- 它们是互补，不是替代
