## 引言

面对任何新的架构提案，问自己这三个问题，可以快速判断其本质和研究价值。

---

## 问题 1：改"连接权重"还是改"建模对象"？

### 核心区分

| 类型 | 定义 | 影响深度 | 例子 |
|------|------|----------|------|
| **连接权重** | 改变信息流动的权重分配方式 | 浅层优化 | Attention Residuals |
| **建模对象** | 改变"什么被建模"的根本假设 | 深层范式 | VAE, Orchestration |

### 具体分析

**改连接权重**的典型特征：
- 保持相同的"层"或"模块"概念
- 改变的是层与层之间的连接方式
- 优化目标通常是训练稳定性或表达能力

**例子 - Attention Residuals**：
```python
# 经典 Residual：固定权重
h_{l+1} = h_l + f(h_l)  # 权重隐式为 1

# Attention Residuals：学习权重
h_{l+1} = Σ α_i · h_i   # α_i = softmax(scores)
```
- 层概念不变
- 改变的是"如何聚合前层输出"
- **判断**：改连接权重，但影响深远

---

**改建模对象**的典型特征：
- 改变"基本单元"的定义
- 引入新的抽象层次
- 优化目标是表示能力或信息压缩

**例子 - VAE**：
```python
# 传统：直接建模 p(x)
# VAE：建模 p(x|z)p(z)，z 是连续潜变量

encoder: x → z (连续潜空间)
decoder: z → x (重构)
```
- 引入"潜变量 z"作为新的建模对象
- 改变的是"什么被优化"
- **判断**：改建模对象，范式级创新

---

## 问题 2：Block 级妥协还是连续化？

### 核心区分

| 类型 | 离散程度 | 工程可行性 | 理想程度 |
|------|----------|------------|----------|
| **Block 级** | 离散 | 高 | 妥协 |
| **半连续** | 混合 | 中 | 探索 |
| **连续** | 连续 | 低 | 理想 |

### 具体分析

**Block 级妥协**：
- 保持离散层或块的结构
- 在块内部或块之间引入灵活性
- 工程上可训练、可部署

**例子 - Attention Residuals**：
- 仍是离散层 l=1,2,3...
- 但第 l 层可以 attention 到任意前层
- **判断**：Block 级妥协，工程可行

---

**半连续（混合）**：
- 试图打破严格的层级结构
- 但仍需要某种离散化以便训练
- 处于探索阶段

**例子 - Perceiver IO**：
- Latent array 是离散的（固定数量 slots）
- 但可以通过 cross-attention 灵活读写
- 类似"离散化的连续记忆"
- **判断**：半连续，有价值但不够彻底

---

**连续化（理想）**：
- 层概念完全消失
- 深度成为连续坐标
- 训练困难，多为理论探索

**例子 - Neural ODE**：
```python
# 连续深度
dh(t)/dt = f(h(t), t, θ)

# 而非离散层
h_{l+1} = h_l + f(h_l)
```
- 深度 t 是连续变量
- 需要 ODE solver
- 训练昂贵，难以 scale
- **判断**：理想但工程困难

---

**我的目标 - CDMN**：
```python
# 愿景：连续深度 + 相位编码检索
h(t) = retrieve(
    memory_bank = {h(0), h(0.1), ...},
    query = current_state,
    addressing = RoPE_angles(t, s)  # 相位编码寻址
)
```
- 深度 t ∈ [0,1] 是连续的
- 但通过相位编码实现离散化检索
- **判断**：试图在连续理想和工程可行之间找平衡

## 问题 3：架构灵感还是产品/安全披露？

### 核心区分

| 类型 | 信息密度 | 研究价值 | 例子 |
|------|----------|----------|------|
| **架构灵感** | 高 | **高** | arXiv 论文, 开源代码 |
| **产品披露** | 中 | 中 | 发布页, benchmark 数字 |
| **安全披露** | 低 | 低 | System Card, 政策文档 |

### 具体分析

**架构灵感（优先阅读）**：
- 来源：arXiv, GitHub, 技术博客
- 内容：方法细节、实现技巧、实验分析
- 价值：可直接借鉴或复现

**例子 - Attention Residuals (arXiv)**：
- 完整的方法描述
- 实验设置和结果
- 开源代码可验证
- **判断**：高价值，优先阅读

---

**产品披露（快速浏览）**：
- 来源：官方发布页、新闻稿
- 内容：能力边界、benchmark 数字、使用场景
- 价值：了解工业界方向，但细节少

**例子 - GPT-5.4 发布页**：
- "比 GPT-4 强 X%"
- 支持 Y 种语言
- 但不告诉你 architecture
- **判断**：低价值（对你当前研究）

---

**安全披露（谨慎对待）**：
- 来源：System Card, Model Card, 政策文档
- 内容：风险评估、安全措施、对齐策略
- 价值：对安全研究有用，对架构创新帮助有限

**例子 - Claude 4.6 System Card**：
- 能力评估
- 安全测试
- 对齐措施
- **判断**：低价值（对你当前架构研究）

## 综合应用：材料筛选实例

### 实例 1：Kimi Attention Residuals

| 问题 | 判断 | 理由 |
|------|------|------|
| 改权重还是对象？ | **权重** | 层概念不变，改变聚合方式 |
| Block 还是连续？ | **Block** | 仍是离散层，可训练部署 |
| 灵感还是披露？ | **灵感** | arXiv 论文，有代码 |

**结论**：高价值，优先阅读，但注意它是 Block 级妥协而非连续化

---

### 实例 2：Perceiver IO

| 问题 | 判断 | 理由 |
|------|------|------|
| 改权重还是对象？ | **对象** | 引入 latent memory 新抽象 |
| Block 还是连续？ | **半连续** | Latent slots 离散，但读写灵活 |
| 灵感还是披露？ | **灵感** | arXiv 论文，DeepMind 出品 |

**结论**：高价值，与 CDMN 思路接近，值得深入

---

### 实例 3：GPT-5.4 System Card

| 问题 | 判断 | 理由 |
|------|------|------|
| 改权重还是对象？ | **未知** | 不披露架构 |
| Block 还是连续？ | **未知** | 同上 |
| 灵感还是披露？ | **披露** | 安全评估文档 |

**结论**：低价值（对你当前研究），快速浏览即可

## 快速决策树

```
新材料出现
    ↓
问题 1: 改权重还是对象?
    ├── 权重 → 优化技术，有价值但非范式
    └── 对象 → 范式创新，深入研究
    ↓
问题 2: Block 还是连续?
    ├── Block → 工程可行，可快速验证
    ├── 半连续 → 探索中，有研究空间
    └── 连续 → 理想但困难，理论价值高
    ↓
问题 3: 灵感还是披露?
    ├── 灵感 (arXiv/代码) → 优先阅读
    ├── 披露 (产品页) → 快速浏览
    └── 安全 (System Card) → 暂缓
```
